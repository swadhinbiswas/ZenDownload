Extensibility

Plugin Developer Guide

Extend ZenDownload with custom plugins. Write a JSON manifest, add hooks, or create full UI pages.

Manifest Format

Place your plugin JSON in the app data plugins directory.

// ~/.local/share/com.zendownload.app/plugins/my-plugin.json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "plugin_type": "webhook",
  "hooks": ["download.complete"]

}

All Manifest Fields

FieldTypeDescription
idstringUnique identifier
namestringDisplay name
versionstringSemVer version
plugin_typestringextractor / postprocessor / webhook / notifier / protocolhandler / mirror / ui
hooksstring[]Events to subscribe to
uiobjectUI manifest (for ui type)

UI Plugins

UI plugins register sidebar entries and render pages using built-in component types. Add a ui block.

{
  "plugin_type": "ui",
  "ui": {
    "sidebar_label": "Radio",
    "component_type": "radio"
  }

}
Radio Player
Built-in station browser with play/pause, Now Playing display. 8 default stations.
RSS Reader
Read RSS/Atom feeds inline. Configurable feed list via page_config.feeds.

Example: Discord Notifier

A webhook plugin that sends a notification to Discord when a download completes.

{
  "id": "discord",
  "name": "Discord",
  "plugin_type": "webhook",
  "hooks": ["download.complete"],
  "config": { "url": "" }

}
Install: Sidebar → Plugins → Browse Store, or manual copy to plugins directory.
