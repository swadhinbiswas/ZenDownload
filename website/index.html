Plugin Developer Guide

Extend ZenDownload with custom plugins. Write a JSON manifest, add hooks, or create full UI pages.

📄 Manifest Format

Place your plugin JSON in the app data plugins directory.

// ~/.local/share/com.zendownload.app/plugins/my-plugin.json
{
"id": "my-plugin",
"name": "My Plugin",
"version": "1.0.0",
"author": "You",
"plugin_type": "webhook",
"enabled": true,
"hooks": ["download.complete"]
}

All Manifest Fields

| Field | Type | Required | Description |
|--------------|----------|----------|------------------------------|
| id | string | yes | Unique plugin identifier |
| name | string | yes | Display name |
| version | string | yes | SemVer version |
| author | string | yes | Creator name |
| description | string | no | Short description |
| homepage | string | no | Project URL |
| plugin_type | string | yes | See types below |
| enabled | boolean | no | Default true |
| hooks | string[] | no | Event hooks to subscribe |
| config | object | no | Default configuration values |
| ui | object | no | UI manifest (for ui plugins) |

🔌 Plugin Types

| Type | Purpose |
|------------------|----------------------------------------|
| extractor | Custom URL parsing/extraction |
| postprocessor | Post-download actions (extract, scan) |
| webhook | Send events to external services |
| notifier | Custom notifications (toast, sound) |
| protocolhandler| Custom protocol:// handling |
| mirror | Alternative mirror detection |
| ui | Register sidebar entries + pages |

Available Hooks

| Hook | Description |
|-----------------------|-----------------------------------------|
| download.start | A new download begins |
| download.complete | A download finishes |
| download.error | A download fails |
| url.extract | URL processing for extraction |
| file.postprocess | After file is downloaded |
| clipboard.detect | Clipboard URL detection |

🎨 UI Plugins

UI plugins register sidebar entries and render pages using built-in component types. Add a ui block to your manifest.

{
"plugin_type": "ui",
"ui": {
"sidebar_label": "Radio",
"sidebar_icon": "Radio",
"component_type": "radio",
"page_config": {}
}
}
Radio Player
component_type: "radio"
Built-in station browser with play/pause, Now Playing display, and stop controls. 8 default stations included.
RSS Reader
component_type: "rss"
Read RSS/Atom feeds inline. Configurable feed list via page_config.feeds.
Custom Page
component_type: "custom"
Render raw HTML or an iframe. Set page_config.html or page_config.iframe_url.

📦 Installation

Plugins can be installed via the in-app Plugin Store or manually.

# Via Plugin Store (recommended)
Sidebar → Plugins → Browse Store → Install

# Manual (Linux/macOS)
mkdir -p ~/.local/share/com.zendownload.app/plugins
cp my-plugin.json "$_"
